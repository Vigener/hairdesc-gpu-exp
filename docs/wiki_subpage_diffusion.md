# 3D 拡散方程式におけるマルチGPU通信隠蔽 (Phase 1-3)

本ページでは、メモリ・通信律速な3D拡散方程式（ステンシル計算）において、Unified Memory（ Managed メモリ）に起因するボトルネックの解消から、非同期通信と計算のオーバーラップによって弱スケーリング効率90%超を達成したマルチGPU最適化のプロセスをまとめる。

---

## 1. 3段階のマルチGPU通信最適化の検証

格子サイズ $1024^3$（約10億格子、約16GBメモリ）の巨大グリッドを用い、複数ノード（1〜8ノード）における弱スケーリング性能を3段階のフェーズに分けて最適化した。

### A. Phase 1: Unified Memory による暗黙転送（アンチスケーリング）
OpenACC の暗黙的な Unified Memory 管理に任せたシンプルな MPI 実装。
- **挙動**: 
  ノード間で境界データ（袖領域）を送受信するたびに、ホスト(CPU)とデバイス(GPU)間で暗黙のページフォルトとページマイグレーションが多発した。
- **結果**: 
  ノード数を増やすほど暗黙転送のレイテンシが蓄積され、スケーリングが著しく悪化する「アンチスケーリング」を招いた。

### B. Phase 2: GPU-Aware MPI による物理転送効率の向上
`host_data use_device` プラグマを用い、GPUメモリのアドレスを直接 MPI 通信関数に渡すことで、暗黙のUnified Memory転送を排除した。
- **挙動**: 
  InfiniBandを介した GPU 間の HBM メモリ直接通信（GPU-Aware MPI）が実現。ページフォルトが解消され、1ノード時の実行時間が **1.71 秒 $\rightarrow$ 0.918 秒** にほぼ半減した。
- **課題**: 
  同期通信（`MPI_Send`/`MPI_Recv`）の待ち時間が露出するため、並列拡張した際に通信オーバーヘッドがそのまま蓄積し、8ノードで実行時間が 1.207 秒に微増した。

### C. Phase 3: 非同期通信と計算のオーバーラップ（弱スケーリング達成）
通信の待ち時間を、計算時間の裏側に隠蔽するための「非同期オーバーラップ」を実装した。
- **手法**:
  1. `MPI_Isend/Irecv` を用いて、境界（袖領域）データの非同期通信を開始。
  2. 通信の完了を待つまでのアイドル時間に、通信データと依存関係のない「格子内部の計算カーネル」を OpenACC の `async` フラグマを用いて非同期に駆動する。
  3. `MPI_Waitall` で通信完了を確認した後、最後に境界（袖領域）の計算カーネルを実行して同期する。
- **結果**: 
  ネットワークの通信レイテンシが計算の実行時間の裏側に隠蔽された結果、8ノード実行時でも **1.027秒 (弱スケーリング効率 90%超)** を維持する良好な拡張性を達成した。

## 2. Nsight Systems による通信隠蔽の実証

2ノード弱スケーリング実行時において、非同期通信と GPU 計算カーネルがどのように並行処理（オーバーラップ）されているかを Nsight Systems によりタイムライン解析し、実証した。

*(※画像配置: `images/nsys_timeline_diffusion_async.png`)*
*図：タイムライン拡大図（MPI_Waitall と GPU 計算カーネルのオーバーラップ）*

- **タイムライン上の実証**:
  CPU 側で非同期送信 `MPI_Isend` が開始された直後、GPU 側で内部領域の計算カーネル `diffusion3d_compute` が即座に起動している。
  CPU は通信完了待ちのために `MPI_Waitall` を呼び出して同期待機（Idle）に入るが、**この `MPI_Waitall` が実行されている時間（$2.83 \text{ ms}$）のほぼ全体にわたり、GPU 側で計算カーネルが並行して走り続けている** ことが視覚的・数値的にも実証された。通信待ち時間が計算によって隠蔽されている。
