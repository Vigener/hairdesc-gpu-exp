# N-body 演算・メモリ最適化のプロセス (Phase 6, 7)

[[Architecture-team/igarashi/Hairdesc_OpenACC]]

#Katex

本ページでは、N-body シミュレーションにおける、グローバルメモリ非合体アクセスの解消（SoA化）および、パイプライン演算ストール（平方根・除算）の排除による最適化プロセスと物理的検証結果をまとめる。

---

## 1. 段階的な最適化アプローチ

### A. データ配置の SoA 化によるメモリアクセス合体化 (Phase 6)
粒子データ構造を、従来の AoS (Array of Structures: 構造体の配列) から SoA (Structure of Arrays: 配列の構造体) へとリファクタリングを施した (`src/nbody_openacc_soa.cpp`)。
- **目的**: 
  N-body ループ内での粒子座標データ（x, y, z）の読込において、隣接スレッドがメモリ上の連続したアドレスに同時にアクセスする「合体メモリアクセス（Coalesced Access）」を達成し、メモリバス帯域の無駄を排除する。
- **結果**: 
  1ノード実行において、実行時間が 6.42 秒から **3.53 秒** へと短縮（**1.82倍高速化**）。H100 GPU の持つ膨大な演算器に対するメモリ供給能力が大幅に向上した。

### B. 手動 `rsqrt` 近似と除算排除 (Phase 7)
SoA化によりメモリ供給のボトルネックを解消した後に露出した、演算器（パイプライン）自体のストールを解消するための最適化。
- **目的**: 
  内ループ内の最大のボトルネックであった倍精度平方根 (`std::sqrt()`) の演算レイテンシとデータ依存、および高負荷な除算処理（約15〜30サイクル）の排除。
- **手法**:
  `const double inv_dist = 1.0 / std::sqrt(dist2);` と記述を一本化し、力学計算中の割り算を乗算へとすべてリファクタリング。コンパイラの `-gpu=fastmath` オプションと連動させることで、GPU に搭載されている高速逆数平方根特殊命令（`rsqrt`）を 100% 発行させ、除算を排除した。
- **結果**: 
  実行時間がさらに 3.53秒 $\rightarrow$ **2.09秒** へと大幅短縮（通常SoA比で **1.69倍高速化**、初期AoS比で **3.08倍高速化**）。

## 2. Nsight Compute によるボトルネック解消の物理的検証

手動最適化による SASS アセンブリレベルでの物理的な変化を、プロファイルデータの Before（改善前）/ After（改善後）の対比により検証した。

### A. 改善前 (Before): `Wait Stalls` が支配的
- 停止要因の **40.70%** が `Wait Stalls`（特殊関数ユニット SFU の平方根演算完了を後続命令が待つためのストール）であり、これが最大の速度向上阻害要因であった。
- 実行時間は **`2.81 ms`**（極小サイズ $N=16384$ での実測）であった。
*(※画像配置: `images/ncu_summary_gpu_soa_before.png`)*

### B. 改善後 (After): ボトルネックの消滅
- **`Wait Stalls` の消滅**: ボトルネック警告一覧から `Wait Stalls` が消失。実行時間は **`1.61 ms`** に激減した。
- **レジスタ削減と Occupancy 向上**: ループ内から高負荷な除算器のレジスタ使用が消えた結果、スレッドあたりのレジスタ数が `82` $\rightarrow$ **`78`** に削減され、SM上のアクティブWarp上限（Theoretical Occupancy）が `31.2%` $\rightarrow$ **`37.5%`** へと向上した。
- **倍精度除算・平方根命令（DDIV/DSQRT）の排除**: 実行された倍精度浮動小数点命令が積和演算 (`DFMA`) と乗算 (`DMUL`) のみになり、倍精度除算 (`DDIV`) および平方根 (`DSQRT`) の発行回数が **ゼロ** になった。
*(※画像配置: `images/ncu_summary_gpu_soa_after.png`)*

## 3. CPU (Grace CPU) への水平展開と大幅な高速化

同様の除算排除リファクタリングを CPU版 (`nbody_hybrid_soa.cpp`) に施した結果、CPU（Miyabi-G Grace, 72T）でも **16.52秒 (1.56倍高速化)** という大幅な性能向上が得られた。

### CPUにおける物理的考察
- **演算遅延自体の解消**: 
  メモリ不連続アクセス（SoA化）によるレイテンシは、CPU の大容量L2/L3キャッシュによって自動的に隠蔽されていた。しかし、**「除算器自体のパイプライン実行遅延（15〜30サイクル）」はキャッシュでは隠蔽不可能な物理ストール** であったため、アルゴリズム側の除算排除がダイレクトに実行時間短縮へと寄与した。
- **SVE2 ベクトル近似命令の稼働**: 
  手動で `inv_dist` を定義したことで、Grace CPU (ARM Neoverse V2) の持つ **SVE2 逆数平方根近似命令（`frsqrte` / `frsqrts`）** が SIMD ベクトル実行ユニット上で 100% 稼働し、パイプラインのストールが解消された。
