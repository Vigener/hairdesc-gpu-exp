# GPUプログラミング学習・実験成果 総括 (Hairdesc_OpenACC)

[[Architecture-team/igarashi]]

#Katex

本ポータルページでは、Miyabi-G (NVIDIA GH200 / H100 GPU & Grace CPU) 上で実施した GPU プログラミング（OpenACC）およびマルチGPU化（MPI+OpenACC）の実践的学習と極限最適化（Phase 1〜7）の成果をまとめている。

詳細な検証結果や時系列のログについては、下部のリンクより各詳細ページを参照のこと。

---

## 1. 目的と概要（何をして、何を学んだか）

- **最適化の実践**:
  N体問題（演算・帯域律速）と3D拡散方程式（メモリ・通信律速）を対象にした、CPU/GPUでの最適化実践。
- **プロファイラ（Nsight Systems / Nsight Compute）による定量的ボトルネック解析**:
  単にコードを書き換えるだけでなく、プロファイラを駆使してカーネル実行内の `Wait Stalls` や `Theoretical Occupancy`、メモリバス帯域利用率、MPI通信タイムラインを解析し、ハードウェアレベルの物理的なストール要因（除算・平方根の遅延や暗黙のページフォルト）を特定して解消するアプローチを習得した。
- **アーキテクチャ特性の学習**:
  SoA化による「メモリ合体アクセス」と、手動rsqrt化による「パイプライン除算遅延の排除」の重要性と、プロセッサ（CPU/GPU）間のレイテンシ隠蔽能力の差異について物理考察を行った。

## 2. 最適化ベンチマーク総括 (N=65536, STEPS=100)

全最適化フェーズを通じて得られた、Miyabi-G 上での CPU (Grace, 72T) および GPU (H100) の絶対性能と累積向上率の推移。

### A. GPU 性能 (H100 GPU)

| 最適化フェーズ | 実行時間 (s) | 実効演算性能 (GFLOPS) | 段階向上率 | AoS初期版からの累積向上率 |
| :--- | :---: | :---: | :---: | :---: |
| **AoS 初期版** (通常 OpenACC) | 6.423 | 1,337 | 1.00x | 1.00x |
| **SoA 通常版** (メモリ合体化) | 3.533 | 2,431 | 1.82倍 | 1.82倍 🚀 |
| **SoA Fast Math手動版** (除算排除) | **2.087** | **4,115** | **1.69倍** | **3.08倍** 🔥 |
| *2ノード並列実行 (SoA Fast Math)* | **1.117** | **7,690** | *スケーリング効率: 93.4%* | **5.75倍** (対1ノードAoS) |

### B. CPU 性能 (Grace CPU, 72スレッド)

| 最適化フェーズ | 実行時間 (s) | 実効演算性能 (GFLOPS) | 段階向上率 | AoS初期版からの累積向上率 |
| :--- | :---: | :---: | :---: | :---: |
| **AoS 初期版** (通常 OpenMP) | 27.79 | 309 | 1.00x | 1.00x |
| **SoA 通常版** (メモリ合体化) | 25.70 | 334 | 1.08倍 | 1.08倍 |
| **SoA Fast Math手動版** (除算排除) | **16.52** | **520** | **1.56倍** | **1.68倍** 🚀 |
| *2ノード並列実行 (SoA Fast Math)* | **8.65** | **993** | *スケーリング効率: 95.5%* | **3.21倍** (対1ノードAoS) |

## 3. テーマ別・詳細検証ストーリー (トピック別サブページ)

### ① [[N-body 演算・メモリ極限最適化の軌跡>./N-body_Optimization]]
- **概要**: 
  Nsight Compute プロファイルに基づき、グローバルメモリ非合体アクセスを SoA 化によって完全解消。さらに、最大ストール要因 `Wait Stalls (40.7%)` の原因であった倍精度平方根 (`std::sqrt`) と除算を手動で排除し、高速な `rsqrt` 近似命令へと再設計することで、GPU で **3.08倍**、CPU で **1.68倍** の限界スループットを引き出した検証ログ。

### ② [[3D 拡散方程式におけるマルチGPU通信隠蔽>./Diffusion3D_Optimization]]
- **概要**: 
  メモリ・通信律速な格子ステンシル計算において、Unified Memory に起因する暗黙ページフォルトのボトルネックを解明。GPU-Aware MPI の適用による転送速度の劇的改善から、`MPI_Isend/Irecv` と OpenACC `async` を用いた非同期の「計算と通信の隠蔽（オーバーラップ）」によって、8ノードで弱スケーリング効率 **90%超** を達成した通信最適化ログ。

### ③ [[CPU vs GPU 性能対照実験の詳細と物理考察>./CPU_vs_GPU_Comparison]]
- **概要**: 
  Miyabi-C (Intel Xeon CPU Max, 112スレッド) および Miyabi-G (Grace CPU, 72スレッド ＋ H100 GPU) を用い、同一演算・同一パラメータ下で絶対性能を直接対照。N-body における L3/HBM3 メモリ・キャッシュ帯域（ルーフライン）での性能律速モデルの検証、および 3D Diffusion でのキャッシュ効果による物理帯域限界突破の実証。
