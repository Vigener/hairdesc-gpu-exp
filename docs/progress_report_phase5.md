# 進捗報告：Phase 5 (3次元拡散方程式 CPU vs GPU 性能対照実験)

## 1. 目的

Phase 4 では、演算律速の傾向が強い N-body シミュレーションを用いて CPU と GPU の性能を比較した。
本フェーズ（Phase 5）では、典型的なメモリ律速および通信律速問題である「3次元拡散方程式（Diffusion3D）」を対象とし、同一条件での CPU 最高性能と GPU 最高性能を定量的に比較・評価する。

特に、Miyabi-G ノードに搭載されている2つのアーキテクチャ：
- **CPU**: NVIDIA Grace CPU (ARM Neoverse V2, 72コア)
- **GPU**: NVIDIA H100 GPU (HBM3 メモリ)
を直接対決させ、計算問題の性質（演算律速 vs メモリ律速）が CPU と GPU の性能差に与える影響を検証する。

---

## 2. 実験環境と問題サイズ

本実験は、同一の Miyabi-G ノード内で完結させることで、ノード間接続やシステムアーキテクチャの差異を排除した純粋なプロセッサ・メモリ比較とした。

- **CPU 版**: MPI + OpenMP ハイブリッド (`diffusion_cpu`, `mpicxx -fast -Mconcur -mp`)
- **GPU 版**: MPI + OpenACC（非同期・通信隠蔽版、`diffusion_gpu`, `mpicxx -fast -acc=gpu -gpu=cc90`)
- **並列構成**:
  - CPU版: 1ノードあたり 1 MPI × 72 OpenMP スレッド
  - GPU版: 1ノードあたり 1 MPI × 1 GPU
- **実験パラメータ (弱スケーリング)**:
  - 1プロセス（ノード）あたりのローカルグリッドサイズ: $1024^3$ (約10億格子点)
  - タイムステップ数: 100 ステップ
  - 1ノードあたりの理論演算量: $13 \text{ FLOP} \times 1024^3 \times 100 \approx 1.39 \times 10^{12}$ FLOP
  - 1ノードあたりのモデル上メモリ転送量: $32 \text{ Bytes} \times 1024^3 \times 100 \approx 3.43 \times 10^{12}$ Bytes (3.43 TB)

---

## 3. 実験結果

### 実行結果（NX=1024, STEPS=100）

| ハード (Miyabi-G) | ノード数 | 実行時間 (s) | 実効演算性能 (GFLOPS) | 実効メモリ帯域 (GB/s) | 1ノード比 |
|---|---|---|---|---|---|
| **CPU** (Grace, 72T) | 1 | 3.855 | 362.1 | 891.4 | 1.00x |
| **CPU** (Grace, 72T) | 2 | 4.231 | 329.9 | 812.1 | 0.91x |
| **GPU** (H100) | 1 | 0.901 | 1,548.6 | 3,811.9 | 1.00x |
| **GPU** (H100) | 2 | 0.909 | 1,536.1 | 3,781.3 | 0.99x |

## 4. 考察

### 1. メモリ律速による CPU vs GPU の性能差の縮小
- 1ノードにおける実効メモリ帯域（BW）を比較すると、GPU が **3,811.9 GB/s**、CPU が **891.4 GB/s** を示し、GPU は CPU の **約 4.3 倍** 高速であった。
- Phase 4 の N-body（演算律速）では GPU が CPU に対して **7.8倍** の性能差をつけていたが、Diffusion3D（メモリ律速）ではその差が **4.3倍** に縮小している。
- これは、Grace CPU の搭載メモリ（LPDDR5X, 理論帯域 512 GB/s）が CPU としては比較的広い帯域を持っていることと、演算に比べてメモリ転送の割合が圧倒的に高いため、GPU の膨大な演算器の優位性が薄れ、純粋なメモリバス幅の勝負（理論帯域比: 3,350 GB/s / 512 GB/s ≈ 6.5倍）に近づいたためと考えられる。

### 2. キャッシュ効果による理論帯域の超過
- 実測された実効メモリ帯域（GPU: 3.81 TB/s, CPU: 891 GB/s）は、それぞれのハードウェアの物理的な理論メモリ帯域（GPU: 3.35 TB/s, CPU: 512 GB/s）を超えている。
- これは、実効メモリ帯域の計算式（1点につき読込7回、書込1回）が「すべてのメモリアクセスがメインメモリ（DRAM/HBM）に到達する」という単純な前提に基づいているためである。
- 実際には、3次元空間の隣接点アクセスにおいて、L2 キャッシュやレジスタ上にデータが保持（再利用）されることで、実際の物理的な DRAM アクセス回数が削減されており、その結果として見かけ上の「実効帯域」が理論物理帯域を上回る結果となった。

### 3. スケーリング特性の差異（非同期通信の効果）
- **GPU 版 (1→2ノード)**: 実行時間は 0.901 秒から 0.909 秒へと、ほぼ横ばい（スケーリング効率 99%）を維持した。これは、Phase 3 で実装した `MPI_Isend/Irecv` による非同期通信と OpenACC の `async` による「通信と計算のオーバーラップ（隠蔽）」が完全に機能していることを示している。
- **CPU 版 (1→2ノード)**: 実行時間は 3.855 秒から 4.231 秒へと、約 10% 増加（スケーリング効率 91%）した。CPU 版は同期的な `MPI_Send/Recv` を用いて境界通信を行っているため、ノード間通信にかかるオーバーヘッドがそのまま実行時間の増加として現れている。

---

## 5. 結論

3次元拡散方程式（メモリ律速問題）における同一 Miyabi-G ノード上の CPU（Grace）と GPU（H100）の性能比較を行った。

1. **実効メモリ性能**: H100 GPU は Grace CPU に対して **約 4.3 倍** 高速であり、演算律速である N-body（約 8 倍）と比較して性能差が大幅に縮小した。メモリ律速な問題ほど、CPU（特に広帯域な Grace や Xeon Max 等）の健闘が際立つことが示された。
2. **通信隠蔽の有効性**: 非同期通信と計算のオーバーラップを適用した GPU 版は、2ノード実行時も 99% のスケーリング効率を達成し、同期通信を用いた CPU 版（91%）に対する並列拡張性の優位性も実証された。
