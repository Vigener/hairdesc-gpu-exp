## 2. N-body シミュレーションにおける対照実験 (Phase 4)

粒子数 $N=65536$, $STEPS=100$ の同一パラメータ下において、CPUスレッド最大実行時と GPU実行時の性能を対照した。

### A. 実測 GFLOPS とピーク比効率
- **Xeon CPU Max (112スレッド)**: 171 GFLOPS（実効ピーク比: **2.5%**）
- **NVIDIA Grace CPU (72スレッド)**: 309 GFLOPS（実効ピーク比: **17.2%**）
- **H100 GPU (AoS版)**: 1,337 GFLOPS（実効ピーク比: **2.0%**）
- **H100 GPU (SoA通常版)**: 2,431 GFLOPS（実効ピーク比: **3.6%**）

### B. 物理考察①：性能差の本質は「演算コア数」ではなく「メモリ帯域」
N体問題は演算量 $\mathcal{O}(N^2)$ の「演算律速」に分類されやすいが、算術強度が $20 \text{ FLOP} / 56 \text{ bytes} \approx 0.36 \text{ FLOP/byte}$ と極めて低く、ルーフラインモデル上では実質的にキャッシュ/メモリ帯域に律速される。
1ノードあたりのメモリ走査時間は、以下の帯域律速モデルで精確に説明可能である。

$$\text{Time} = \frac{N^2 \times 56\text{ bytes}}{\text{Effective Bandwidth}}$$

- **Xeon MAX**: 有効帯域 $\sim 500\text{ GB/s}$ (L3有効) $\rightarrow$ 予測 **48秒**（実測: 50.10秒 ✅）
- **H100 GPU**: 有効帯域 $3,350\text{ GB/s}$ (HBM3) $\rightarrow$ 予測 **7.2秒**（実測: 6.42秒 ✅）

物理メモリ帯域幅の比（3.35 TB/s / ~0.5 TB/s $\approx$ **6.7倍**）が、1ノード実測 GFLOPS 比（AoS時 **7.8倍**）とほぼ一致しており、**GPU の性能優位性の本質は、演算コアの多さではなく、物理的なメモリ帯域の圧倒的な広さである** ことが実証された。

### C. 物理考察②：Grace CPU の卓越した効率と拡張性
- Grace CPU は Xeon MAX に対して **約1.8倍高速** であり、ピーク比効率 **17.2%** という極めて高い数値を記録した。
  これは、Xeon MAX が 112 スレッドのアクセスによる L3 キャッシュ競合（バス詰まり）を引き起こしたのに対し、Grace CPU (72コア) はコア間キャッシュ競合が最小限に抑えられ、メモリ・キャッシュ帯域を極めて効率的に引き出せたためである。
- 2ノード Strong Scaling において、Xeon MAX は同期通信オーバーヘッドによりスケーリング効率が **81.0%** に低下したが、Grace CPU はプロセスバインド解除と低レイテンシインターコネクトの恩恵により、**97.5%** という GPU（98.1%）と同等の完璧なスケーリングを示した。

## 3. 3D 拡散方程式における対照実験 (Phase 5)

格子サイズ $1024^3$、100ステップの弱スケーリング条件において、同一ノード（Miyabi-G）上の Grace CPU（72スレッド）と H100 GPU を直接対戦させた。

### A. 実測結果 (1ノード)
- **H100 GPU**: 実行時間 **0.901 秒** (実効メモリ帯域: **3.81 TB/s**)
- **Grace CPU**: 実行時間 **3.855 秒** (実効メモリ帯域: **891.4 GB/s**)
- **性能差**: GPU は CPU に対して **約4.3倍** 高速（N-body の 8倍差から縮小）。

### B. 物理考察③：メモリ律速問題における性能差の縮小とキャッシュ効果
- **メモリ帯域勝負へのシフト**:
  3D Diffusion は算術強度が極めて低く、GPU 側の膨大な演算コアの優位性が発揮されづらいため、純粋なメモリ帯域幅の勝負（理論帯域比: 3.35 TB/s / 512 GB/s $\approx$ 6.5倍）にシフトし、絶対性能差が縮小した。
- **キャッシュ効果による物理限界の突破**:
  実測された実効メモリ帯域（GPU: 3.81 TB/s, CPU: 891 GB/s）は、双方の物理的な理論メモリ帯域（H100: 3.35 TB/s, Grace: 512 GB/s）を大きく上回っている。
  これは、3次元ステンシル計算の隣接格子アクセスにおいて、データがオンチップキャッシュ（L2/L3）やレジスタ上に高確率でヒットして再利用されたため、物理 DRAM (HBM3/LPDDR5X) へのトランザクションが大幅に削減された（物理限界を上回る見かけ上の帯域幅として現れた）ことを実証している。
