# 進捗報告：Phase 1 (マルチGPU化・Unified Memory検証)

## 1. 目的
HAIRDESCの講義資料に基づき、OpenACCによるGPU化済みのコード（N体問題、3次元拡散方程式）をMPIを用いてマルチGPU化する。
初期実装として、実装コストが最も低い **「Managed / Unified Memory」** を利用した手法（単一GPUコードのMPI並列化と同等）を採用し、スパコン環境でのスケーリング性能とアーキテクチャ特性を評価することを目的とした。

## 2. ターゲット問題と実装内容
- **N体問題 (N-body)**
  - 特性: 計算量が $\mathcal{O}(N^2)$、通信量が $\mathcal{O}(N)$ となる**演算律速**の代表例。
  - 実装: `src/nbody_openacc.cpp`
- **3次元拡散方程式 (Diffusion3D)**
  - 特性: 空間を格子状に分割し、隣接格子（袖領域）のデータ交換を行う**メモリ律速・通信律速**の代表例。
  - 実装: `src/diffusion3d_openacc.cpp`

両コードとも、OpenACCの `#pragma acc managed` (Unified Memory) オプションを前提に実装し、GPU⇔CPU間のデータ移動管理をコンパイラ・ランタイム（NVHPC）に一任する「最もシンプルなマルチGPU化」を行っている。

## 3. 実験環境と問題サイズ
- **計算機環境**: 東京大学 / 筑波大学 共同利用スパコン「Miyabi-G」 (NVIDIA Grace CPU + H100 GPU / GH200)
- **ノード数**: 1, 2, 4, 8 ノード（1ノードあたり 1 MPIプロセス / 1 GPU を割り当て）
- **実験した問題サイズ**:
  - N-body: **N = 65,536**（中規模サイズ、計算負荷が十分に高い）
  - Diffusion3D: **NX = 1,024**（1024x1024x1024 = 約10億グリッド、約16GBメモリ消費）

## 4. 実験結果と考察

### N体問題のスケーリング結果 (N=65536)
| ノード数 (GPU数) | 実行時間 (秒) | 速度向上率 (対1ノード) |
|---|---|---|
| 1 | 63.46 | 1.00x |
| 2 | 32.40 | 1.96x |
| 4 | 16.50 | 3.84x |
| 8 | 8.50 | 7.46x |

**【考察】**
演算律速であるN-body問題においては、Unified Memoryを用いた単純なMPI通信実装であっても、計算時間が圧倒的に通信時間を上回るため、**ほぼ理想的な線形スケーリング（Strong Scaling）**を達成できることが証明された。

### 3次元拡散方程式のスケーリング結果 (NX=1024)
| ノード数 (GPU数) | 実行時間 (秒) | 速度向上率 (対1ノード) |
|---|---|---|
| 1 | 1.71 | 1.00x |
| 2 | 1.82 | 0.94x |
| 4 | 1.91 | 0.89x |
| 8 | 2.09 | 0.81x |

**【考察】**
10億グリッド（16GB）という巨大サイズであっても、GH200（H100）の数TB/sに及ぶ異次元のメモリ帯域の前では、計算自体が1.7秒で完了してしまう。一方で、ノードを追加すると実行時間が逆に増加する**アンチスケーリング**が観測された。
これは、**Unified Memory (Managed) を介したMPI通信のボトルネック**が原因である。データ送信時、GPUメモリからCPUメモリへ暗黙の移動（ページフォルト）が発生し、そこからInfiniBand等のネットワークを経由し、受信側で再度CPUからGPUへとデータが移動する。この「CPU⇔GPU間の往復＋ネットワークレイテンシ」のオーバーヘッドが、極限まで高速化されたGPUの内部計算時間を上回ってしまったためである。

## 5. 次のステップへの示唆
以上の結果から、メモリ律速のステンシル計算（Diffusion3D）をマルチGPUでスケールさせるためには、単なるUnified Memory任せの実装では限界があることが実証された。
講義資料の中級編（マルチGPUプログラミング）で示されている通り、これを解決するための次なる実装として、**GPU-Aware MPI**（`#pragma acc host_data use_device` プラグマを用いてGPU上のアドレスを直接MPIに渡し、暗黙のメモリ移動を排除する手法）の導入が必須となる。
