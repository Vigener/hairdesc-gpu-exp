# 進捗報告：Phase 4 (CPU vs GPU 性能対照実験)

## 1. 目的

これまでのフェーズ（Phase 1〜3）はすべて **Miyabi-G（NVIDIA H100 GPU）** 上での実験であった。
本フェーズでは、Miyabi クラスタが持つ**2つの異なる計算ノード**を対象として、同一のN体問題を同一条件で実行し、「CPUの最高性能」対「GPUの最高性能」を定量的に比較する。

具体的には、N体シミュレーションの実装を以下の2系統に分類する：
- **CPU版** (`nbody_cpu_fast`): MPI + OpenMP ハイブリッド実装。Miyabi-C (Intel Xeon CPU Max) で実行。
- **GPU版** (`nbody_gpu_fast`): MPI + OpenACC 実装。Miyabi-G (NVIDIA H100 / GH200) で実行。

## 2. 実験環境と問題サイズ

| | CPU 版 | GPU 版 |
|---|---|---|
| **実行クラスタ** | Miyabi-C | Miyabi-G |
| **プロセッサ** | Intel Xeon CPU Max 9480 × 2S | NVIDIA H100 (GH200) |
| **並列構成** | 1 MPI × 112 OMP threads / ノード | 1 MPI × 1 GPU / ノード |
| **コンパイラ** | Intel oneAPI `mpiicpx -O3 -qopenmp -axSAPPHIRERAPIDS,CORE-AVX512` | NVIDIA HPC SDK `mpicxx -fast -acc=gpu -gpu=cc90` |
| **ジョブキュー** | `short-c` | `debug-g` |

- **問題サイズ**: N = 65,536 粒子（2のべき乗。1, 2ノードへの分割が整数で割り切れる）
- **ステップ数**: STEPS = 100
- **総演算量 (1 ノード)**: $20 \times N \times (N-1) \times \text{STEPS} \approx 8.59 \times 10^{12}$ FLOP = 8.59 TFLOP

## 3. 実験結果

### Strong Scaling 結果

| ハード | ノード数 | MPI / OMP | 実行時間 (s) | GFLOPS | 対1ノード比 | スケーリング効率 |
|---|---|---|---|---|---|---|
| **CPU** (Xeon MAX) | 1 | 1 MPI × 112 OMP | 50.10 | 171 | 1.00x | – |
| **CPU** (Xeon MAX) | 2 | 2 MPI × 112 OMP | 30.85 | 278 | **1.62x** | **81%** |
| **GPU** (H100) | 1 | 1 MPI × 1 GPU | 6.43 | 1,336 | 1.00x | – |
| **GPU** (H100) | 2 | 2 MPI × 2 GPU | 3.31 | 2,595 | **1.94x** | **97%** |

### CPU vs GPU 性能比較

| ノード数 | GPU GFLOPS | CPU GFLOPS | **GPU / CPU 性能比** |
|---|---|---|---|
| 1 ノード | 1,336 | 171 | **7.8倍** |
| 2 ノード | 2,595 | 278 | **9.3倍** |

### カタログスペックとの乖離

| | **CPU (Xeon MAX, 1 node)** | **GPU (H100, 1 node)** |
|---|---|---|
| **理論ピーク DP** | 6,720 GFLOPS (6.72 TFLOPS) | 67,000 GFLOPS (67 TFLOPS) |
| **実測 GFLOPS** | **171 GFLOPS** | **1,336 GFLOPS** |
| **ピーク比効率** | **2.5%** | **2.0%** |

## 4. 考察

### 1. H100 は Xeon MAX の約8倍高速（N-body O(N²) 演算律速問題において）
1ノード同士の比較では、GPU が CPU の **7.8倍**の GFLOPS を達成した。
これは H100 (67 TFLOPS 理論) vs Xeon MAX (6.72 TFLOPS 理論) の理論比 **9.95倍**と比較して妥当な数値である。
どちらも `sqrt()` がボトルネックであるため理論ピークの利用率は同水準（約 2〜2.5%）にとどまるが、
絶対的な演算器数とメモリ帯域でH100が圧倒しているため、実行 GFLOPS は約8倍の差として現れる。

### 2. GPU のスケーリング効率がほぼ理想的（1.94x / 2 ノード）
- **GPU (1→2ノード)**: 1.94x = 効率 97%。N=65,536 の規模では各 GPU に 32,768 粒子が割り当てられ、H100 の膨大な並列コア数に対して十分な負荷が存在するため、ほぼ線形スケーリングが達成できた。これは Phase 1 の GPU-only スケーリング実験の結果（8ノードで線形スケーリング）と一致する。
- **CPU (1→2ノード)**: 1.62x = 効率 81%。MPI_Allgather による同期通信のオーバーヘッドが全体の 19% 程度を占めており、GPU に比べてスケーリング効率がやや落ちる。ただし81%は実用的に十分な値である。

### 3. CPU の実行 GFLOPS は問題サイズ依存（小N vs 大N）
N=5,000 時の CPU 実測は **389 GFLOPS** であったが、N=65,536 では **171 GFLOPS** へ低下した。
これは、N=5,000 では粒子配列（約 280 KB）が L2/L3 キャッシュに収まるのに対し、N=65,536 では配列サイズが約 3.67 MB となり、112 スレッドが同時にこの配列を参照することでキャッシュ帯域の競合が発生するためと考えられる。
GPU はメモリ構造（HBM3）とスレッド隠蔽機能により、大N時もキャッシュ帯域の競合が緩和されるため、GFLOPS の低下が相対的に小さい。

### 4. 過去の GPU-only 実験との整合性確認
Phase 1 の GPU スケーリング実験（N=65,536, STEPS=1000）では：
- 1 GPU: 63.46 秒 → STEPS=100 換算: **6.35 秒**（本実験の実測値 6.43 秒と一致 ✅）
- 2 GPU: 32.40 秒 → STEPS=100 換算: **3.24 秒**（本実験の実測値 3.31 秒と一致 ✅）

結果の再現性が高く、実装・実験環境の信頼性が確認された。

## 5. 結論

N体問題（$\mathcal{O}(N^2)$ 演算律速）において、Miyabi クラスタが提供する以下のアーキテクチャの性能を実測した：

| | 実測 GFLOPS | 理論比効率 | 対 CPU 優位性 |
|---|---|---|---|
| **CPU** (Xeon MAX, 1 ノード) | 171 GFLOPS | 2.5% | – |
| **GPU** (H100, 1 ノード) | 1,336 GFLOPS | 2.0% | **7.8 倍** |

`sqrt()` ボトルネックにより両者ともピーク効率は約2%と低いが、GPU は絶対的な演算器数の優位性により CPU を**約8倍**上回る実行性能を示した。
また GPU は Strong Scaling において 97% という高い並列効率を達成し、マルチノード構成での拡張性においても CPU（81%）を凌駕した。
